This class provides a wrapper for any module to ensure module related generic events like loaded, unloaded, data sent. IMPORTANT: This class is still confusing and little buggy code! load()method
public function load():void

The loadfunctio is seperated from the constructor to ensure that the parent object can add Event Listeners to this module wrapper class. The load function seperated between the parameter object=null or not. If not it will load the module with the class definition stored in the parameter. --> getDefinitionByName(parameter). If its null it will load the login module. This represents an easy logic: on start parameter is null so it will load the login and after login the mainappmanager will# handle which specific module has to be load.
