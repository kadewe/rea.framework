PackageTracking
Classpublic class InputTracker
InheritanceInputTracker Object

The InputTracker Class provides static methods for logging user input. Its underlying strucutre is an internal model, see InputTrackerModel, which buffers the log entries until the sendLog is called. The model is also a dynamic object which is refreshed autmatically, once the old log is send to the server.

logEvent()method
public static function logEvent(currentView:String, mousePosX:uint, mousePosY:uint, currentEvent:String, eventTarget:String):void

Adds a log entry to the internal log model. Beware that null objects are logged with a null entry instead of empty strings.

Parameters

currentView:String — The current state of a model's view component. Use the returnCurrentState() Method, implemented by ICustomModule get the the actual viewstate
 
mousePosX:uint — allows you to log the mouseposition x axis as a unsigned integer
 
mousePosY:uint — allows you to log the mouseposition Y axis as a unsigned integer
 
currentEvent:String — Does not need to be an eventType. You may also write things like "click" or "module start" or whatever you might want to log.
 
eventTarget:String — Does not need to be an even target Object (if so, use event.target.id if possible). You may also write things like "Start Button" or whatever you want to log.

sendLog()method 
public static function sendLog(url:String, navigate:Boolean = false):void

Sends the log to the given url via URLRequest. If navigate is true, the results can be viewed in a new browser window. The data is passed in the following structure: param = "tracking" (To let php indicate which script to include). type = String(model.returnType()) id = String(model.returnUser()) (In this case id means userid) xml = model.returnFinal()(The final xml content to be send)
